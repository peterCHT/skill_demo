# meeting-summary

這是一個最小可用的Skill demo，用來示範如何把一段會議逐字稿、零散會議筆記或站會更新，整理成結構化的會後摘要。

這個 skill 會把輸入內容整理成：

- 摘要
- 決議
- 待辦事項
- 風險與待釐清問題

## Demo 重點

這個範例刻意保持簡單，只包含一個 `SKILL.md`，不需要 API key、不需要額外腳本，也不需要任何外部依賴。適合用來展示：

- skill 的基本資料夾結構
- `SKILL.md` 的 frontmatter 寫法
- 如何用中文描述 skill 的觸發條件與工作流程
- 安裝後如何用 `$meeting-summary` 觸發 skill

## 資料夾結構

```text
meeting-summary/
  SKILL.md
  README.md
```

## 使用方式

1. 可以在直接在任意AI agent對話框中輸入 "請幫我安裝這個技能 https://github.com/peterCHT/skill_demo"

2. 安裝這個 skill 後，可以在AI agent中輸入：

```text
Use $meeting-summary 整理這段會議筆記：

Alice 說登入頁本週五要上線。Bob 還在等設計稿。Carol 說 API 已經 ready。
決定先不上 SSO，留到下個 sprint。風險是 QA 時間只剩兩天。
Bob 要明天下午前補設計稿。
```

預期輸出會包含四個區塊：

```text
## 摘要

- 登入頁目標是在本週五上線。
- API 已經準備完成。
- 設計稿仍在等待 Bob 補齊。

## 決議

- SSO 先不上線，延到下個 sprint。

## 待辦事項

- Bob：補設計稿，明天下午前

## 風險與待釐清問題

- QA 時間只剩兩天，可能影響上線品質或時程。
```

## 設計說明

這個 demo 展示的核心概念是：Skill 不一定要包含複雜工具或程式碼。即使只有一份 `SKILL.md`，也可以把一套固定的工作方式封裝起來，讓 Codex 在類似任務中穩定地按照同一種格式與標準輸出。

# Part 2｜初學 Skill：白話解釋小幫手

這一部分示範如何建立一個簡單、容易觸發，也方便觀察執行結果的 `SKILL.md`。

範例任務是：當使用者表示看不懂某個概念時，用繁體中文重新解釋，並固定提供一句話說明、生活比喻與簡單例子。

## 為什麼選擇這個範例

- 功能單純，一眼就能看懂 Skill 要做什麼。
- 不需要程式、外部資料或額外素材。
- 可以直接用自然語言測試是否成功觸發。
- 適合練習 `description`、Workflow、輸出格式與驗證規則。

## 資料夾結構

```text
explain-simply/
└── SKILL.md
```

這個範例只需要 `SKILL.md`。

不需要為了看起來完整，額外建立沒有實際用途的 `scripts/`、`references/` 或 `assets/`。


## 這份 Skill 的執行順序

```text
使用者提出問題
→ description 判斷是否符合觸發情境
→ 讀取 SKILL.md
→ 找出主要概念
→ 寫一句話解釋
→ 加入生活比喻
→ 加入簡單例子
→ 完成前檢查
→ 交付結果
```

## Description 怎麼寫

這份 description 包含三個部分：

1. **功能**：向沒有背景的初學者解釋單一概念。
2. **固定產物**：一句話定義、生活比喻與簡單例子。
3. **明確觸發句**：列出使用者實際可能輸入的語句。

觸發句包括：

- `白話解釋`
- `簡單解釋`
- `簡單說`
- `我看不懂`
- `我是初學者`
- `用小學生也懂的方式`
- `什麼是 X`
- `X 是什麼意思`

Skill 並不是只進行完全一致的關鍵字比對，但加入這些具體語句，可以讓模型更容易判斷使用時機。

## Workflow 怎麼寫

Workflow 是 Skill 被觸發後的執行路線。

這個範例的 Workflow 很短：

```text
辨識概念
→ 一句話解釋
→ 生活比喻
→ 簡單例子
→ 檢查難懂名詞
```

每個步驟只交代一個動作，因此容易閱讀，也不需要模型猜測執行順序。

最後再補上一個簡單分支：如果無法辨識概念，就先詢問使用者，不要直接猜測。

## 測試方式

建立 Skill 後，可以使用以下句子測試：

```text
請白話解釋 API。
```

```text
我是初學者，什麼是資料庫？
```

```text
OAuth 是什麼意思？用小學生也懂的方式說明。
```

```text
這段我看不懂，請簡單說明什麼是雲端運算。
```

檢查輸出是否同時包含：

- 一句話解釋
- 生活比喻
- 簡單例子
- 繁體中文
- 250 字以內

## 初學者可以學到什麼

- `description` 要同時寫功能、產物與觸發情境。
- Workflow 要按照真正的執行順序排列。
- 輸出格式要直接規定成品長什麼樣子。
- 完成前檢查要使用可以驗證的條件。
- 簡單 Skill 不需要硬加不會使用的資源資料夾。
